Signup form referrer URL

Recall: site attributes

  • API supports storing arbitrary key/value pairs for each site.
  • Filtering/sorting by attributes will be introduced soon.

Use case

  • On signup form, we also provide a distinct signup.form.uri value as site attribute.
  • This identifies the form URI used for submission.
  • Different sources of submission use different values, e.g.
    • https://signup.beyondshop.cloud
    • https://mate.beyondshop.cloud
    • https://epages.elavon.com
    • ...

Utilization

  • Clients (like MATe 3.0) with access to site service will be able to filter all sites by this referrer.
  • More arbitrary attributes can be added as meta data on site provisioning, like
    • mate.authorized.user=tsenechal
    • mate.auto.delete=2018-03-01T00:00:00Z
    • reseller=SAGE

Matomo

Decimal numbers as custom attributes

  • Introduced new attribute definitions type Number
  • With an option to specify a number format
  • Number supersedes Long and Double* formats
  • Migrated existing numeric types to Number

Attribute Definition Creation

POST https://.../product-attribute-definitions
{
  "namespace" : "custom",
  "name" : "filling_capacity",
  "displayName" : "Filling Capacity",
  "type" : "NUMBER",
  "format" : "#,###.## L"
}
