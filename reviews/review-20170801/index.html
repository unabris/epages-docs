Automated tests for Docker Image builds

Motivation

  • increasing complexity of ng-cluster-gateway
  • confidence in making big changes

Goals

  • integration tests using the real docker images
  • easy debuggability
  • generic structure to use as a template

Preparatory Work

  • create/adjust local docker-compose environment
  • evaluate test framework & write tests
  • enabling jenkins to run test setup

Jenkins Job Structure

1. docker-build.sh
builds TIMESTAMP
2. docker-test.sh
tests TIMESTAMP (or derived)
3. docker-push.sh
pushes TIMESTAMP latest
4. docker-cleanup.sh
removes build artifacts

docker-test

  • docker-compose enabled
  • produces:
    • log file(s)
    • TAP file(s)

Test Anything Protocol

Example

1..4
ok     1 - Input file opened
not ok 2 - First line of the input valid
# line did not match the given regular expression
ok     3 - Read the rest of the file
not ok 4 - Summarized correctly # SKIP feature disabled
