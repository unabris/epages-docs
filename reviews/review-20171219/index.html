Piwik

Requirements

  • Tracking a typical customer journey
  • Provide the merchant shop statistics
  • Automated onboarding of new merchants
  • PM analytics tool

Tracking a typical customer journey

  • Comprehensive tracking facilities for frontend events
  • Build-in e-commerce specific events that can be tracked
  • Possible to track async server-side events
  • Correlation of tracked frontend and backend data

Provide the merchant shop statistics

  • Provide shop statistics widgets embedded in merchant-ui
  • No option to style widgets, responsiveness is not perfect
  • Option to obtain raw data to create custom charts

Automated onboarding of new merchants

  • Piwik has a multi-user / multi-website concept
  • It's possible to create users and sites using the API

PM analytics tool

  • Offers a concept of aggregated data from multiple sites
  • Piwik UI can be used as PM analytics tool

Other findings

  • The piwik API is old-fashioned and a bit quirky
  • But it works and supports many use-cases
  • Piwik had an outage on the first day we were using it
  • Was fixed pretty quickly

Site Attributes

Requirements driven by MATe 3.0

  • Distinguish sites of different resellers
  • Distinguish developers sites
  • Provide flexible implementation ready for changes

Solution

Introduce custom string attributes to ng-site

Managed via REST API

  • POST /sites/{uuid}/attributes/
    
{
  "name"  : "reseller",
  "value" : "La Poste",
}

    
HTTP/1.1 201 Created
...
Location: .../sites/{uuid}/attributes/

  • DELETE /sites/{uuid}/attributes/reseller
    
HTTP/1.1 204 No Content
...

Communicate via messages to other services


{
    "messageType" : "site.site-attribute.created.event",
    "timestamp" : "2017-12-15T14:53:19.319",
